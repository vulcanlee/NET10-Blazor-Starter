# 認證授權與權限機制

## 文件目的

說明本專案如何處理「登入」、「身分判斷」與「功能授權」三件事，包含 Cookie 認證、Claims 結構、`RoleView` 權限樹、`Menu.json` 對應，以及 Sidebar 的動態隱藏。

> 本文聚焦「機制總覽」，「為何重新開站還在登入」之原理請見 [記住我登入原理說明](記住我登入原理說明.md)。

---

## 1. 認證方案

採 ASP.NET Core 內建 **Cookie Authentication**，於 [`src/MyProject/MyProject.Web/Program.cs:101-108`](../src/MyProject/MyProject.Web/Program.cs) 註冊：

```csharp
builder.Services.AddAuthentication(MagicObjectHelper.CookieScheme)
    .AddCookie(MagicObjectHelper.CookieScheme, options =>
    {
        options.Cookie.IsEssential = true;
        options.LoginPath = "/Auths/Login";
        options.LogoutPath = "/Auths/Logout";
        options.AccessDeniedPath = "/Auths/Login";
    });
```

關鍵設定：

| 設定 | 用途 |
|------|------|
| `Cookie.IsEssential = true` | 標記為必要 Cookie，避免被 Cookie 同意機制阻擋。 |
| `LoginPath = "/Auths/Login"` | 未登入時導向此路徑。 |
| `LogoutPath = "/Auths/Logout"` | 登出走的路徑。 |
| `AccessDeniedPath = "/Auths/Login"` | 權限不足時的導向（亦回登入頁）。 |

## 2. 登入流程

檔案：`MyProject.Web/Components/Auths/Login.razor` + `Login.razor.cs`

1. 使用者輸入「帳號 / 密碼 / 4 位驗證碼」並可勾選「記住我」。
2. 後端比對驗證碼、帳密；密碼採 `PasswordHelper.GetPasswordSHA(salt, password)` 與資料庫雜湊比對。
3. 通過後組成 Claims：`Role`、`Name`、`NameIdentifier`、`Sid`。
4. 建立 `AuthenticationProperties { IsPersistent = Input.RememberMe }`。
5. 呼叫 `HttpContext.SignInAsync(MagicObjectHelper.CookieScheme, ...)`。

驗證碼錯誤或登入失敗時會重新產生新的驗證碼，UI 詳情見 [Login 頁面改版紀錄](login-redesign.md)。

---

## 3. 登出流程

檔案：`MyProject.Web/Components/Auths/Logout.razor.cs`

呼叫 `HttpContext.SignOutAsync(MagicObjectHelper.CookieScheme)` 移除驗證票證；之後再進站需重新登入。

## 4. 取得目前使用者：`AuthenticationStateHelper`

檔案：`MyProject.Business/Services/Other/AuthenticationStateHelper.cs`

主要工作：

1. 透過 `AuthenticationStateProvider.GetAuthenticationStateAsync()` 取得 `ClaimsPrincipal`。
2. 檢查 `IsAuthenticated`，從 `ClaimTypes.Sid` 取使用者 Id。
3. 由 Id 從資料庫取出完整 `MyUser`、解析 `RoleView.TabViewJson`。
4. 將結果寫入 `CurrentUserService.CurrentUser`，供整個元件樹共享。

進入需驗證的頁面前統一呼叫 `AuthenticationStateHelper.Check(...)`，因此各 Razor 元件不需要自行 boilerplate 認證邏輯。

## 5. 角色與權限樹

### 5.1 `RoleView` Entity

| 欄位 | 用途 |
|------|------|
| `Name` | 角色顯示名稱（例：管理員、檢視者）。 |
| `TabViewJson` | 角色擁有的權限項目，以 JSON 序列化後存放。 |

### 5.2 二維結構

`TabViewJson` 反序列化後型別為 **`List<List<string>>`**：

- 第一層每個 `List<string>` 對應 Sidebar 的「子功能表名稱」（例：`["專案管理", "專案項目", "工作項目"]`）。
- 對於沒有子功能表的單層項目（例如「首頁」、「登出」），`List<string>` 只會包含一個成員。

實際清單由 `RolePermissionService` 產生，方法為 `GetRoleListPermissionAllName()`，與 `Menu.json` 結構互相對應。

### 5.3 RoleView CRUD UI

`MyProject.Web/Components/Views/Admins/RoleViewView.razor` 中以巢狀 Checkbox 顯示二維結構，第一層為功能表名稱、第二層為該功能下的頁面權限。

## 6. Sidebar 與 `Menu.json`

檔案：[`src/MyProject/MyProject.Web/Datas/Menu.json`](../src/MyProject/MyProject.Web/Datas/Menu.json)

範例：

```json
[
  { "id": 1, "name": "首頁", "icon": "space_dashboard", "url": "/App" },
  {
    "id": 2, "name": "專案管理", "icon": "folder_managed",
    "subMenu": [
      { "id": 21, "name": "專案項目",  "icon": "workspaces", "url": "/projects" },
      { "id": 22, "name": "工作項目",  "icon": "checklist",  "url": "/Task" },
      { "id": 23, "name": "會議記錄",  "icon": "event",      "url": "/meeting" }
    ]
  },
  ...
  { "id": 4, "name": "登出", "icon": "logout", "url": "/auths/logout" }
]
```

每個 `name` 即為「權限名稱」。要判斷使用者是否能看到某個功能：

```csharp
AuthenticationStateHelper.CheckAccessPage(MagicObjectHelper.角色_會議項目)
```

`SidebarMenuService` 會在組裝 Sidebar 時將每個節點都跑一次此檢查，無權限的節點會被隱藏。

`icon` 欄位採 Google Material Icons 名稱（搜尋於 https://fonts.google.com/icons），透過 `BlazorMaterialIcons` 套件渲染。

## 7. 預設帳號與角色 Seed

`Program.cs:175-258` 啟動時會自動：

1. 確保 `RoleView.Name == MagicObjectHelper.預設角色` 存在，其 `TabViewJson` 永遠覆寫為「全部權限 JSON」。
2. 確保 `MyUser.Account == MagicObjectHelper.開發者帳號` 存在，`IsAdmin = true`，密碼為帳號名稱。
3. 將開發者帳號的 `RoleViewId` 對應到上述預設角色。

> Seed 行為冪等。每次啟動都會覆寫預設角色權限與開發者密碼，方便重置。

## 8. 常見問題

| 問題 | 原因 / 對策 |
|------|------|
| 登入後重新整理就被踢回登入頁 | 確認 `Cookie.IsEssential = true` 仍存在；可能被 Cookie 同意政策擋下。 |
| 持久化 Cookie 未生效 | 檢查 `IsPersistent = Input.RememberMe` 是否被正確讀取，詳見 [記住我登入原理說明](記住我登入原理說明.md)。 |
| 新增功能但 Sidebar 看不到 | 確認 `Menu.json` 已新增節點、且使用者角色的 `TabViewJson` 包含對應 `name`。 |
| 開發者帳號密碼忘記 | 重啟程式即可重置，密碼等同 `MagicObjectHelper.開發者帳號`。 |
