# 檔案上傳機制

## 文件目的

說明專案、工作、會議三種紀錄的附件如何上傳、儲存、刪除，以及實體檔案的目錄組織策略。

---

## 1. 三類附件對照

| 主表 | 附件 Entity | 設定檔欄位（`SystemSettings.ExternalFileSystem`） |
|------|--------------|----------------------------------------------------|
| `Project` | `ProjectFile` | `ProjectFilePath` |
| `MyTask`（DbSet 名 `MyTas`） | `MyTasFile` | `TaskFilePath` |
| `Meeting` | `MeetingFile` | `MeetingFilePath` |

預設值（`appsettings.json`）：

```json
"ProjectFilePath":  "C:\\temp\\MyProject\\ProjectFile",
"TaskFilePath":     "C:\\temp\\MyProject\\TaskFilePath",
"MeetingFilePath":  "C:\\temp\\MyProject\\MeetingFilePath"
```

啟動時 `Program.cs:132-141` 會呼叫 `EnsureDirectoryExists(...)` 自動建立缺少的目錄。

---

## 2. 目錄組織策略

每筆紀錄的附件依「主表的建立時間」決定目錄分層：

```
{各模組根目錄}/
└── {年}/                ← 主表 CreatedAt.Year
    └── {月}/             ← 主表 CreatedAt.Month（補零兩碼）
        ├── 檔案 A
        ├── 檔案 B
        └── ...
```

例：2026 年 5 月新增的專案，其檔案會落在 `C:\temp\MyProject\ProjectFile\2026\05\` 下。

---

## 3. 上傳行為

- 一筆紀錄可上傳多個檔案。
- 單檔最大 1GB（前端與後端皆需驗證）。
- 在「新增模式」下，使用者上傳的檔案會先暫留於 UI；按下「儲存」後，Service 才將實體檔案落地到對應年月目錄，並建立資料庫紀錄。
- 在「修改模式」下：
  - 既有檔案會列在清單中，可手動移除（標記為待刪除）。
  - 可繼續加入新檔案，落地行為同新增模式。
  - 按下「儲存」後：被標記為待刪除的紀錄連同實體檔案一併移除；新檔案則寫入目錄並登入資料庫。

---

## 4. 刪除行為

當使用者刪除主表紀錄（例如刪除一筆專案）：

1. EF Core 因 `DeleteBehavior.Cascade`（見 `BackendDBContext.OnModelCreating`）會自動刪除附件資料表的 row。
2. **實體檔案**仍須由 Service 層在刪除前蒐集 `StoredPath`、刪除主表後再呼叫 `File.Delete(...)`。
3. 若實體檔案刪除失敗（例如已被外部移動），Service 應寫入 `Warning` 等級日誌，但不阻斷主表刪除流程。

---

## 5. 對應 DTO 與設定強型別

`SystemSettings.ExternalFileSystem` 透過 `IOptions<SystemSettings>` 注入到 Service：

```csharp
public class FooService(IOptions<SystemSettings> options, ILogger<FooService> logger)
{
    private readonly ExternalFileSystem fileSystem = options.Value.ExternalFileSystem;
    // ...
}
```

於 Service 內統一使用 `fileSystem.ProjectFilePath` 等屬性，**禁止**直接從 `IConfiguration` 讀字串。

---

## 6. `/UploadFiles` 靜態資源對應

`Program.cs:319-325`：

```csharp
app.UseStaticFiles(new StaticFileOptions
{
    FileProvider = new PhysicalFileProvider(systemSettings.ExternalFileSystem.DownloadPath),
    RequestPath = "/UploadFiles"
});
```

- `DownloadPath` 是「公開可下載」目錄；任何放在這裡的檔案都會被 `/UploadFiles/...` 對外服務。
- 三類附件目錄（`ProjectFilePath`、`TaskFilePath`、`MeetingFilePath`）**不**直接對應 `/UploadFiles`；下載這些附件需透過 Controller 端點，由 Service 驗證權限後再回傳檔案串流。

---

## 7. 實作檢查清單

新增「附件功能」時請確認：

- [ ] Entity 已新增 `XxxFile` 子表，並在 `BackendDBContext.OnModelCreating` 設定 `Cascade`。
- [ ] `appsettings.json` 已新增對應根目錄欄位，並在 `SystemSettings.ExternalFileSystem` 模型中對映。
- [ ] `Program.cs:132-141` 已呼叫 `EnsureDirectoryExists` 包含新目錄。
- [ ] Service 對外提供 `Add` / `Update` / `Delete` 時皆同步處理實體檔案。
- [ ] 對外下載端點需做角色檢查與檔名清洗，避免路徑跳脫攻擊。
