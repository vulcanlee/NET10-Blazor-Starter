# 多語系與本地化

## 文件目的

說明本專案如何處理 .NET 在地化、瀏覽器語系自動偵測，以及 AntDesign Blazor 元件的本地化字串覆寫。

---

## 1. 支援文化

目前內建兩種文化：

| Culture | 顯示 | 預設 |
|---------|------|------|
| `zh-TW` | 繁體中文（台灣） | ✅ 預設 |
| `en-US` | 英文（美國） | |

## 2. .NET RequestLocalization

```csharp
var supportedCultures = new[] { new CultureInfo("zh-TW"), new CultureInfo("en-US") };
var defaultCulture = supportedCultures[0];

builder.Services.Configure<RequestLocalizationOptions>(options =>
{
    options.DefaultRequestCulture = new RequestCulture(defaultCulture);
    options.SupportedCultures = supportedCultures;
    options.SupportedUICultures = supportedCultures;
    options.RequestCultureProviders = new List<IRequestCultureProvider>
    {
        new AcceptLanguageHeaderRequestCultureProvider()
    };
});
```

關鍵點：

- 採用 `AcceptLanguageHeaderRequestCultureProvider`，**根據瀏覽器 `Accept-Language` 標頭自動切換文化**。
- 移除預設的 Cookie / Query 文化提供者，避免使用者手動鎖定文化。
- 中介軟體啟用：`app.UseRequestLocalization(localizationOptions);`（`Program.cs:310-313`）。

> 若日後要支援使用者手動切換語系（例如下拉式選單），需另行加入 Cookie / Query Provider，並提供切換 UI。

## 3. AntDesign 元件本地化

AntDesign Blazor 內建多語字典，但部分字串（例如表格排序、過濾、確認）需要客製。本專案以 `AntDesignLocaleFactory` 集中覆寫：

```csharp
LocaleProvider.SetLocale("zh-TW", AntDesignLocaleFactory.Create("zh-TW"));
LocaleProvider.DefaultLanguage = defaultCulture.Name;
```

檔案：`src/MyProject/MyProject.Web/Localization/AntDesignLocaleFactory.cs`

工廠職責：

1. 取得 AntDesign 內建 `Locale`（依 culture 名稱）。
2. 覆寫需要客製的字串（例：表格欄位排序提示、確認對話框預設文字）。
3. 回傳客製後的 `Locale` 物件交給 `LocaleProvider`。

新增覆寫項目時請集中於此檔，避免散落於各元件。

---

## 4. 在元件中使用本地化

### 4.1 使用 `IStringLocalizer<T>`

於需要翻譯的元件以 DI 注入：

```razor
@inject IStringLocalizer<MyComponent> Localizer

<h1>@Localizer["Welcome"]</h1>
```

對應字串需放在 `Resources/MyComponent.zh-TW.resx`、`Resources/MyComponent.en-US.resx`（資料夾結構需與元件命名空間對齊）。

> 本範本目前以「即時硬編碼字串」為主，若要全面導入 IStringLocalizer，請於 `MyProject.Web/Resources/` 下建立資源檔。

### 4.2 AntDesign 元件自動套用

`<Table>`、`<DatePicker>`、`<Modal>` 等元件會自動使用 `LocaleProvider.CurrentLocale`，無須手動傳入 `Locale` 屬性。

## 5. 文化切換驗證

可用瀏覽器設定變更語系驗證：

1. Chrome → 設定 → 語言 → 將想測試的語言（例如 English (United States)）拉到第一順位。
2. 重新整理頁面，AntDesign 表格、按鈕等預設字串會切換到 `en-US` 對應翻譯。

或者以 PowerShell 直接送出帶 `Accept-Language` 標頭的請求：

```powershell
Invoke-WebRequest -Uri "https://localhost:5001/" -Headers @{ "Accept-Language" = "en-US" }
```
