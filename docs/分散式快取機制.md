# 分散式快取機制

## 文件目的

說明本專案的可設定快取層：如何透過 `appsettings.json` 在記憶體與 Redis 之間切換、`ICacheService` 的用法，以及第一個實際使用者（側邊選單）的快取行為與失效方式。

---

## 1. 設計概念

快取統一建立在 .NET 內建的 `IDistributedCache` 之上，因此記憶體與 Redis 兩種實作對呼叫端完全一致——**切換 provider 不需修改任何使用端程式**。

- `Provider=Memory`：使用 `AddDistributedMemoryCache`，零外部相依，適合單機與開發。
- `Provider=Redis`：使用 `AddStackExchangeRedisCache`，適合多節點部署時的「全域共享快取」。

切換邏輯比照資料庫 provider 切換（`ServiceCollectionExtensions.AddConfiguredDatabase`），集中在 [`ServiceCollectionExtensions.AddConfiguredCache`](../src/MyProject/MyProject.Web/Extensions/ServiceCollectionExtensions.cs)，並於 `Program.cs` 接線。

---

## 2. 設定

檔案：[`src/MyProject/MyProject.Web/appsettings.json`](../src/MyProject/MyProject.Web/appsettings.json)

```jsonc
"CacheSettings": {
  "Provider":                 "Memory",   // Memory | Redis
  "RedisConnection":          "",         // Provider=Redis 時必填，例如 localhost:6379
  "InstanceName":             "MyProject:",// Redis 鍵前綴
  "DefaultExpirationMinutes": 30           // 未指定過期時的預設絕對過期（分鐘）
}
```

| 欄位 | 說明 |
|------|------|
| `Provider` | 空字串回退為 `Memory`；未知值在啟動時拋出 `InvalidOperationException`（容錯解析見 `CacheSettings.GetProvider()`）。 |
| `RedisConnection` | `Provider=Redis` 時不可為空白，否則 `AddConfiguredCache` 直接拋例外。 |
| `InstanceName` | 多個應用共用同一 Redis 實例時用來隔離鍵空間。 |
| `DefaultExpirationMinutes` | 最小值為 1；`SetAsync` / `GetOrCreateAsync` 未指定 `expiration` 時套用此值。 |

### Production 安全檢查

[`StartupSafetyValidator`](../src/MyProject/MyProject.Web/Configuration/StartupSafetyValidator.cs) 在 Production 啟動時檢查：若 `Provider=Redis` 但 `RedisConnection` 留空，立即阻擋啟動並回明確錯誤訊息。

## 3. `ICacheService` 用法

介面：[`ICacheService`](../src/MyProject/MyProject.Web/Caching/ICacheService.cs)，實作：[`DistributedCacheService`](../src/MyProject/MyProject.Web/Caching/DistributedCacheService.cs)（以單例註冊）。內含 JSON 序列化，呼叫端只處理強型別物件。

```csharp
// 取值（未命中回 default）
var menu = await cacheService.GetAsync<List<MenuItem>>("some:key");

// 設值（不指定過期時套用 DefaultExpirationMinutes）
await cacheService.SetAsync("some:key", menu, TimeSpan.FromMinutes(10));

// 取或建：未命中時呼叫 factory 並回填快取
var menu = await cacheService.GetOrCreateAsync(
    "some:key",
    () => LoadFromSourceAsync());

// 主動失效
await cacheService.RemoveAsync("some:key");
```

> **限制**：`GetOrCreateAsync` 以「`null` 代表未命中」判定，適用參考型別。若日後需快取數值型別（如 `int`），需另行設計命中判定。

## 4. 第一個使用者：側邊選單

[`SidebarMenuService`](../src/MyProject/MyProject.Web/Components/Layout/SidebarMenuService.cs) 原本每次載入選單都從磁碟讀取並反序列化 `Menu.json`。現在改為：

- 以鍵 `sidebar:menu:raw` 快取**原始選單反序列化結果**（檔案靜態內容）。
- 依使用者權限的 `ApplyPermissionStructure` / `FilterAuthorizedMenuItems` **維持每次計算、不快取**，因此不同使用者仍看到正確的授權選單。

### 失效行為

`Menu.json` 變更後，需待快取過期（預設 30 分）或應用重啟才會生效；若要即時生效，可呼叫 `RemoveAsync("sidebar:menu:raw")`。一般而言 `Menu.json` 屬隨版本部署的靜態結構，此行為可接受。
