# 測試指南

本專案以 **xUnit** 撰寫單元與整合測試，測試專案為 `src/MyProject/MyProject.Tests`，並納入 CI 自動執行（見 [CI-CD 與品質檢查](../operations/CI-CD與品質檢查.md)）。

---

## 1. 測試框架與套件

`MyProject.Tests` 使用：

- **xUnit** — 測試框架（`[Fact]` / `[Theory]`）。
- **Microsoft.AspNetCore.Mvc.Testing** — 以 `WebApplicationFactory` 啟動整個 Web API 做整合測試。
- **Microsoft.NET.Test.Sdk** — 測試執行器。
- **coverlet.collector** — 程式碼覆蓋率蒐集。

實際版本以 `src/MyProject/MyProject.Tests/MyProject.Tests.csproj` 為準。

---

## 2. 測試類別與涵蓋範圍

| 測試類別 | 類型 | 涵蓋重點 |
|----------|------|----------|
| `ApiIntegrationTests` | 整合 | 端到端驗證 API：無 Bearer 回 401、login/refresh/me、`/api/v1` 版本化合約、validation 400、403、500、`/health/live`、`/health/ready`、`system-health` 未登入不外洩、Production 安全檢查 fail-fast、`DatabaseProvider` 解析。 |
| `ApiResultTests` | 單元 | `ApiResult<T>` 信封：驗證錯誤保留結構化 errors、伺服器錯誤包含例外資訊。 |
| `JwtTokenServiceTests` | 單元 | JWT：產生 access/refresh token、驗證 refresh token 取回使用者、拒絕以 access token 當 refresh。 |
| `MyUserServicePasswordTests` | 單元 | 變更密碼：正確舊密碼才更新、錯誤舊密碼／空白／確認不符不更新、support 帳號不可改。 |
| `AuthenticationStateHelperTests` | 單元 | `AuthenticationStateHelper.Check`：未登入、Sid 無效、使用者不存在／停用、缺 RoleView、角色 JSON 無效、強制改密碼例外頁、快取仍重檢、合法使用者初始化 `CurrentUser`。 |
| `CacheServiceTests` | 單元 | `ICacheService`：複雜物件序列化往返、Miss 回預設值、`GetOrCreateAsync` factory 只跑一次、`RemoveAsync` 失效。 |
| `CacheSettingsTests` | 單元 | `CacheSettings`：空白預設 Memory、大小寫不敏感解析 Redis、未知值丟例外。 |
| `SystemHealthTests` | 單元 | 健康分數：全綠 100、退化黃燈、不健康紅燈、燈號對應、`HealthLogReader` 讀最後 100 行、檔案不存在回退。 |

## 3. 在本機執行

```powershell
# 執行整個方案的所有測試
dotnet test src/MyProject/MyProject.slnx

# 只跑測試專案
dotnet test src/MyProject/MyProject.Tests/MyProject.Tests.csproj

# 以單一測試類別或方法過濾
dotnet test src/MyProject/MyProject.slnx --filter "FullyQualifiedName~ApiIntegrationTests"
dotnet test src/MyProject/MyProject.slnx --filter "Name=GetProvider_ShouldThrow_WhenUnknown"
```

CI 以 Release 組態執行：`dotnet test src/MyProject/MyProject.slnx --configuration Release --no-build`（先經 restore/build 階段）。

---

## 4. 覆蓋率

`coverlet.collector` 已隨測試專案引入，可於執行時蒐集覆蓋率：

```powershell
dotnet test src/MyProject/MyProject.slnx --collect:"XPlat Code Coverage"
```

報表（Cobertura XML）會輸出至各測試專案的 `TestResults/` 下，可再以報表工具轉檢視。

## 5. 撰寫新測試的慣例

- 整合測試走 `WebApplicationFactory`，驗證對外 **`ApiResult` 合約**（狀態碼 + 信封結構），不依賴內部實作細節；需觸發 403／500 等情境時，透過測試專案的 probe controller 製造，**不新增正式 API contract**。
- 單元測試針對 Service / Helper 的純邏輯，必要時以 in-memory 或測試替身隔離資料庫。
- 新增 / 修改 API 時，請補對應整合測試；新增 Entity 或密碼／權限邏輯時，補單元測試。對照 [維護規範](../operations/維護規範.md) 的 commit 前檢查清單。
