# 密碼種類與儲存機制

## 目的

本文件彙整腳手架專案中所有與「密碼」相關的種類、儲存位置與處理方式，釐清哪些是真正逐筆儲存於資料庫的密碼、哪些是設定檔中的密碼或常數，以及哪些屬於機密金鑰（嚴格說不是帳號密碼）。

## 總覽

### 真正的「密碼」

| 種類 | 儲存位置 / 欄位 | 形式 |
| --- | --- | --- |
| 使用者帳號密碼 | 資料庫 `MyUser.Password` + `MyUser.Salt` | SHA256 雜湊（每人不同鹽） |
| 啟動種子管理員密碼 | 設定檔 `BootstrapSettings:SupportPassword` | 明碼（預設 `support`） |
| 預設 / 需變更密碼常數 | 程式常數 `MagicObjectHelper.NeedChangePassword` | 明碼常數（`123456`） |

### 機密金鑰（非帳號密碼，但常被歸類為「密碼」）

| 種類 | 儲存位置 |
| --- | --- |
| JWT 簽章金鑰 `JwtSettings:SigningKey` | `appsettings.json`（建議以 user-secrets / 環境變數覆寫） |
| Google OAuth 用戶端密鑰 `GoogleOAuthSettings:ClientSecret` | `appsettings.json`（支援 user-secrets / 環境變數） |

## 1. 使用者帳號密碼（核心）

整個系統中，唯一以資料表欄位形式逐筆儲存的密碼就是 `MyUser.Password`，搭配 `MyUser.Salt`。

- 資料表欄位：`src/MyProject/MyProject.AccessDatas/Models/MyUser.cs`
  - `Password`：SHA256 雜湊後的字串，不存明碼。
  - `Salt`：每位使用者一組隨機鹽（`Guid.NewGuid().ToString()`）。
- 記憶體傳輸用鏡像：`MyUserAdapterModel.Password` / `.Salt`（`src/MyProject/MyProject.Models/AdapterModel/MyUserAdapterModel.cs`），僅在傳輸/編輯時暫存，不落地保存。

### 雜湊機制

密碼雜湊由 `PasswordHelper.GetPasswordSHA(salt, password)` 產生（`src/MyProject/MyProject.Business/Helpers/PasswordHelper.cs`）：

```
hash = SHA256( "{password}-{salt}@" )   // 轉為小寫十六進位字串
```

> 備註：`PasswordHelper.GetGodPasswordSHA` 與 `GodSalt版本` 定義了「萬用密碼」機制，但目前全專案沒有任何呼叫點（dead code），保留供未來判斷是否使用。

### 密碼的三種來源，皆收斂到同一欄位

`MyUser.Password` 同時涵蓋以下三種情境，最後都以雜湊形式寫入同一欄位：

1. **一般本地帳號登入密碼** — 管理者於使用者管理建立 / 變更密碼時雜湊寫入。
2. **啟動種子管理員密碼** — `support` 帳號於系統啟動時種子建立（見第 2 節）。
3. **Google OAuth 使用者的「API 密碼」** — Google 帳號於 `/Profile` 自行設定，用來取得 JWT（見下一節）。

## 2. Google 使用者的「API 密碼」

「API 密碼」並不是新欄位。Google OAuth 帳號預設沒有本地密碼，無法直接呼叫需要 JWT 的 Web API；使用者可在 `/Profile` 頁面設定一組密碼，**以 SHA256 雜湊寫入資料庫的同一個 `MyUser.Password` 欄位**，之後即可用「帳號 + 此密碼」換取 JWT。

實際寫入由 `MyUserService.SetOwnApiPasswordAsync(...)` 完成（`src/MyProject/MyProject.Business/Services/DataAccess/MyUserService.cs`）：

```csharp
MyUser? user = await context.MyUser.FirstOrDefaultAsync(x => x.Id == userId);
...
user.Salt = string.IsNullOrWhiteSpace(user.Salt) ? Guid.NewGuid().ToString() : user.Salt;
user.Password = PasswordHelper.GetPasswordSHA(user.Salt, newPassword);
user.UpdateAt = DateTime.Now;
await context.SaveChangesAsync();   // 持久化到資料庫
```

流程重點：

- **是否寫入資料庫**：是，透過 `context.SaveChangesAsync()` 持久化。
- **儲存欄位**：`MyUser.Password`（+ `MyUser.Salt`），與一般本地密碼完全共用。
- **儲存形式**：SHA256 雜湊，非明碼。
- **首次設定（Google 帳號）**：`user.Password` 為空 → 免驗舊密碼，並用 `Guid.NewGuid()` 產生新的 `Salt`。
- **已設過密碼者**：需通過 `currentPassword` 雜湊比對才能變更。
- **限制**：系統預設開發帳號 `support`（`MagicObjectHelper.開發者帳號`）禁止於此變更密碼。

## 3. 登入與 JWT 驗證

無論是 UI 帳號密碼登入或 API 換取 JWT，皆比對 `MyUser.Password`：

- 比對邏輯：`MyUserServiceLogin.LoginAsync(username, password)`（`src/MyProject/MyProject.Business/Services/Other/MyUserServiceLogin.cs`）。
- 將輸入密碼以該使用者的 `Salt` 重新雜湊，與資料庫中的 `Password` 比對，相同才視為通過。

## 4. 啟動種子管理員密碼（SupportPassword）

- 設定欄位：`appsettings.json` 的 `BootstrapSettings:SupportPassword`（預設 `support`），對應類別 `SystemSettings.cs` 的 `BootstrapSettings.SupportPassword`。
- 用途：系統啟動時種子建立 `support` 管理帳號，於 `Program.cs` 取得此設定值後，一樣以 `GetPasswordSHA` 雜湊後寫入 `MyUser.Password`。
- 安全防護：`StartupSafetyValidator` 會在 Production 環境阻擋使用預設值 `support`，避免正式環境沿用弱密碼。

## 5. 預設 / 需變更密碼常數（NeedChangePassword）

- 常數：`MagicObjectHelper.NeedChangePassword = "123456"`（`src/MyProject/MyProject.Share/Helpers/MagicObjectHelper.cs`）。
- 用途：作為新帳號初始密碼；`MyUserService.NeedChangePasswordAsync(...)` 會比對使用者目前密碼雜湊是否仍等於此預設值，藉此在登入後提示 / 強制使用者變更密碼（導向 `/ChangePassword`）。

## 6. 機密金鑰（非帳號密碼）

以下屬於系統機密，與帳號密碼性質不同，存於設定檔而非資料表：

- `JwtSettings:SigningKey`：JWT 簽章金鑰，正式環境務必以 user-secrets 或環境變數覆寫，勿沿用範本預設值。
- `GoogleOAuthSettings:ClientSecret`：Google OAuth 用戶端密鑰，支援 user-secrets / 環境變數。

## 結論

實際以資料表欄位逐筆儲存的密碼只有一個 —— `MyUser.Password`（+ `Salt`），且一律為 SHA256 雜湊。一般登入、種子管理員、Google 使用者的 API 密碼三者皆收斂到此欄位；`SupportPassword`、`NeedChangePassword` 是來源 / 常數，而 `SigningKey`、`ClientSecret` 則屬於機密金鑰。
