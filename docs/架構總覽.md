# 架構總覽

## 文件目的

說明本專案的整體分層、各專案職責、依賴方向、啟動流程，以及 DI 容器中註冊的主要服務，協助新進開發者快速建立心智模型。

---

## 1. 解決方案組成

方案檔：`src/MyProject/MyProject.slnx`（採新版 `.slnx` 格式）。共 6 個專案，採典型的分層架構：

| 專案 | 主要職責 | 依賴方向 |
|------|----------|----------|
| `MyProject.Web` | Blazor Server 宿主、Razor 元件、Web API、登入、本地化、靜態資源 | 依賴所有其他專案 |
| `MyProject.Business` | 商業邏輯：Service、Repository、AutoMapper Profile、Helper | → AccessDatas、Models、Dtos、Share |
| `MyProject.AccessDatas` | EF Core `DbContext`、Entity、Migration | → 無外部專案相依 |
| `MyProject.Models` | AdapterModel、System 設定模型、共用領域模型 | → 無外部專案相依 |
| `MyProject.Dtos` | API 對外 DTO：`ApiResult<T>`、`PagedResult<T>`、模組 DTO | → 無外部專案相依 |
| `MyProject.Share` | 跨層共用 Helper、Extension（純工具） | → 無外部專案相依 |

> 依賴方向永遠由「上層」指向「下層」，不可反向引用。Web 不應跳過 Business 直接操作 `BackendDBContext`，但 Web 內的 API Controller 為了輕量化，使用 `Repository`（仍位於 Business 專案）。

## 2. 啟動流程（`MyProject.Web/Program.cs`）

啟動主檔位於 [`src/MyProject/MyProject.Web/Program.cs`](../src/MyProject/MyProject.Web/Program.cs)。整體流程：

1. **建立 `WebApplicationBuilder`**（`Program.cs:31`）。
2. **NLog 設定**（`Program.cs:33-56`）：
   - 從 `NLog:BasePath` 讀取根目錄，於其下建立 `MyProject.Web` 子目錄。
   - 清除預設 logger provider，改由 `UseNLog()` 接管。
   - 立即建構服務取得 `ILogger<Program>` 寫入啟動日誌。
3. **核心服務註冊**（`Program.cs:58-92`）：
   - `AddRazorComponents().AddInteractiveServerComponents()`：Blazor Server 互動式渲染。
   - `AddControllers()`、`AddSwaggerGen()`：傳統 Web API + Swagger。
   - `AddLocalization()`、`AddAntDesign()`：在地化與 AntDesign。
   - `RequestLocalizationOptions`：支援 `zh-TW` / `en-US`，採 `AcceptLanguageHeaderRequestCultureProvider`。
   - `LocaleProvider.SetLocale("zh-TW", AntDesignLocaleFactory.Create("zh-TW"))`：套用本地化字串。
4. **認證**（`Program.cs:94-110`）：
   - Cookie 政策：`CheckConsentNeeded = true`、`MinimumSameSitePolicy = None`。
   - `AddAuthentication(MagicObjectHelper.CookieScheme).AddCookie(...)`：登入 / 登出 / 拒絕路徑均指向 `/Auths/Login` 系列。
   - `AddAuthorization()`。
5. **AutoMapper**（`Program.cs:112-123`）：載入 `AutoMapping` Profile，可選擇套用 LicenseKey。
6. **強型別設定**（`Program.cs:128-130`）：將 `SystemSettings` section 綁定為 `IOptions<SystemSettings>`。
7. **外部目錄準備**（`Program.cs:132-141`）：依 `SystemSettings.ExternalFileSystem` 各路徑呼叫 `EnsureDirectoryExists`，缺少時自動建立。
8. **EF Core**（`Program.cs:143-153`）：使用 `UseSqlite(...)` 註冊 `BackendDBContext`，連線字串由 `MagicObjectHelper.GetSQLiteConnectionString` 結合 `DatabasePath` 動態產生。
9. **客製服務**（`Program.cs:155-169`）：所有 Service / Repository 一律 `Scoped` 生命週期。
10. **資料庫初始化**（`Program.cs:175-258`）：
    - 若有 Migration 則 `Database.Migrate()`，否則 `EnsureCreated()`。
    - Seed 預設 `RoleView`（`MagicObjectHelper.預設角色`）並寫入完整權限 JSON。
    - Seed 預設 `MyUser`（`MagicObjectHelper.開發者帳號`），密碼以 SHA + Salt 存放，標記為 `IsAdmin`。
11. **中介軟體管線**（`Program.cs:260-333`）：
    - 自訂 middleware 紀錄每筆 HTTP 請求耗時與例外。
    - `UseStatusCodePagesWithReExecute("/not-found")`、`UseHttpsRedirection`、`UseRequestLocalization`、`UseAntiforgery`。
    - `MapStaticAssets`、`UseStaticFiles` 將 `DownloadPath` 對應為 `/UploadFiles`。
    - `UseAuthentication` → `UseAuthorization`。
    - `MapControllers`、`MapRazorComponents<App>().AddInteractiveServerRenderMode()`。

## 3. 已註冊的 Scoped 服務

於 `Program.cs:155-169` 一次列出，便於對照：

```
AuthenticationStateHelper      // 取得目前登入狀態 + 載入 CurrentUser
CurrentUserService             // 全局保存目前登入使用者（含角色 / 權限）
MyUserServiceLogin             // 登入專用驗證流程（密碼比對、驗證碼）
SidebarMenuService             // 載入 Menu.json 並依角色過濾
RolePermissionService          // 角色權限樹（GetRoleListPermissionAllName 等）
RoleViewService                // 角色檢視（RoleView Entity）CRUD
MyUserService                  // 使用者 CRUD
ProjectService / ProjectRepository
MyTaskRepository / MyTasService
MeetingRepository / MeetingService
```

> Repository 用於 Web API（`Controllers/`），Service 用於 Razor 元件 / Page；兩者目前並存。

## 4. 請求生命週期

### 4.1 Razor 元件請求（Blazor Server）

```
Browser ─► UseAuthentication ─► UseAuthorization ─► MapRazorComponents
                                                         │
                                                         ▼
                                         Layout.razor + Page.razor
                                                         │
                                                         ▼
                                       View 元件 ──► Service ──► DbContext
```

`AuthenticationStateHelper.Check(...)` 會在進入受保護頁面時讀取 `ClaimsPrincipal`、由 `Sid` 對應到 `MyUser`，並把使用者寫入 `CurrentUserService.CurrentUser` 供整個元件樹共享。

### 4.2 Web API 請求

```
Browser ─► UseAuthentication ─► UseAuthorization ─► MapControllers
                                                         │
                                                         ▼
                                       Controller ──► Repository ──► DbContext
                                                         │
                                                         ▼
                                       回傳 ApiResult<T> 或 PagedResult<T>
```

詳見 [Web API 設計慣例](Web%20API%20設計慣例.md)。

## 5. 程式碼分層慣例

| 慣例 | 說明 |
|------|------|
| 命名對齊 | `XxxPage` / `XxxView` / `XxxService` / `XxxRepository` / `XxxAdapterModel` / `XxxDto` |
| UI 不直接觸 DbContext | Razor 只呼叫 Service / Repository，不直接 `using` `BackendDBContext`。 |
| Edit 前先 Clone | 表格列被編輯前要 `Clone()` 隔離，避免雙向綁定污染來源。 |
| EF 追蹤清理 | Update / Delete 前以 `CleanTrackingHelper` 清除追蹤實體。 |
| 通知統一走 AntDesign | 成功 / 失敗一律經 `NotificationService` / `MessageService`。 |
| API 回應統一 | 任何 Controller 一律回傳 `ApiResult<T>`，分頁則包 `PagedResult<T>`。 |

詳細模板請見 [建立一個新 CRUD 操作網頁說明](建立一個新%20CRUD%20操作網頁說明.md)。
