# 新增「分類清單」與「團隊清單」管理頁面（0.3.0）

## 變更目的

本腳手架將逐步成長為知識庫萃取系統，以母專案 KnowledgeExtraction.AI 為藍本分階段移植功能。**階段一**新增兩個獨立的主資料管理頁面：分類清單（Category）與團隊清單（Team），作為後續「紀錄分類/團隊標籤 + 團隊權控」（階段二）的前置基礎。

兩者為獨立主資料（無外鍵關聯），本階段不修改 Project/Task/Meeting/Role 等既有頁面。

## 功能範圍

- **分類清單**：名稱（必填、唯一）、描述、啟用狀態；CRUD、搜尋、排序、分頁。
- **團隊清單**：名稱（必填、唯一）、代號（選填、有填則唯一）、描述、啟用狀態；CRUD、搜尋、排序、分頁。
- **Web API**：`CategoryController` / `TeamController`，提供 GetById / Search / Create / Update / Delete，回傳 `ApiResult<T>`，JWT Bearer 授權，`/api` 與 `/api/v1` 平行路由。
- **權限**：新增「資料定義」選單群組，子項「分類清單」「團隊清單」，納入角色權限樹。

- **權限位置索引**：`Menu.json` 與 `RolePermissionService.GetRoleListPermissionAllName()` 以位置對應。新群組插在「登出」之前，保留既有群組索引不變；預設角色於啟動時自動重新 seed 同步，**既有自訂角色需重新勾選**新權限。
