# 側邊欄群組圖示依名稱各自顯示（0.4.4）

## 變更目的

側邊欄（收合與展開）中，所有「群組」節點原本都顯示同一個資料夾圖示，無法分辨。本次讓每個群組依其名稱顯示各自合適、且彼此相異的圖示。

## 根因

`NavMenu.razor.cs`（收合）與 `SidebarMenuNode.razor.cs`（展開）的 `GetMaterialIconKind` 都有 `_ when item.HasChildren => "folder_open"`，把所有群組節點的 icon 一律覆寫成 `folder_open`，導致 `Menu.json` 內各群組設定的 icon 被忽略。（子項不受影響，故飛出子項圖示原本就正確。）

附帶：`專案管理` 原 `folder_managed` 為 Material Symbols 專有名稱，不在 App 載入的 classic Material Icons 字型（與 0.2.63 的 `database` 同類），即使移除覆寫也會破圖。

## 本次修改

- `NavMenu.razor.cs`、`SidebarMenuNode.razor.cs`：移除 `_ when (item/Item).HasChildren => "folder_open"`，群組改用 `Menu.json` 自己的 icon（僅當 icon 空白才回退 `folder_open`）。展開與收合 icon 解析自此一致。
- `Datas/Menu.json` 群組 icon 改為有效且各異的 classic Material Icons：
  - 專案管理：`folder_managed` → `work`
  - 資料定義：`dataset` → `storage`
  - 系統管理：`manage_accounts` → `admin_panel_settings`
  - （首頁 `space_dashboard`、登出 `logout`、各子項 icon 不變）
- `MenuIconTests.cs`：允許清單同步（移除 `folder_managed`/`dataset`/`manage_accounts`，新增 `work`/`storage`/`admin_panel_settings`）。
- `appsettings.json`：`SystemVersion` `0.4.3 → 0.4.4`。

## 驗證

- `dotnet build -c Release`：0 錯誤；`dotnet test`：80 通過（含更新後的 MenuIconTests）。
- 手動（需重建 + Ctrl+F5）：收合與展開側邊欄，三個群組圖示各異且正常渲染——專案管理=公事包、資料定義=資料庫、系統管理=盾牌人。
