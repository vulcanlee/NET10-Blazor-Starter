# 紀錄分類/團隊標籤與團隊權控（0.4.0）

## 變更目的

階段二：把階段一建立的「分類（Category）」「團隊（Team）」主資料，掛到 Project / MyTask / Meeting 三大紀錄成為多值標籤，並導入以角色為基礎的團隊行級權控——**非管理員只能看到「無團隊（公開）」或「與其角色團隊交集」的紀錄**。做法移植自母專案 KnowledgeExtraction.AI。

## 功能範圍

- **標籤**：Project/MyTask/Meeting 各新增 `Categories`、`Teams`（多值，以換行分隔字串儲存）。
- **清單**：三頁面工具列新增「分類過濾」「團隊過濾」多選；表格新增分類/團隊欄。
- **編輯**：三頁面編輯表單新增分類/團隊多選。
- **角色**：RoleView 新增 `DefaultTeamsJson`；角色管理頁可指派「預設團隊」。
- **權控**（Blazor 服務層，同母專案）：
  - 清單 `GetAsync` 對非管理員套團隊可見述詞；單筆 `GetAsync(id)` 與檔案下載以 `IsTeamAccessible` 守門。
  - 登入時把角色預設團隊載入 `CurrentUser.TeamList`。
  - Web API（repository 路徑）維持 JWT 保護，**不**做行級過濾（與母專案一致）。

## 設計重點

- **儲存格式**：標籤以 `"\n值1\n值2\n"` 包夾儲存，查詢用 `Contains("\n值\n")` 做精確成員比對（避免子字串誤判）。見 `TagStringHelper`。
- **存取範圍**：`IRecordAccessScopeProvider` / `RecordAccessScopeProvider` 雙模式解析（Blazor 用 `CurrentUserService`、其他情境讀 HttpContext claims）。
- **轉換**：AutoMapper `ForMember` 在 Entity（分隔字串）↔ AdapterModel（`List<string>`）間轉換；RoleView 的 `DefaultTeams` ↔ `DefaultTeamsJson` 以 `TeamJsonHelper`。

## 注意事項

- **既有資料**：升級後既有紀錄的 `Teams` 為 null（公開），管理員與所有角色皆可見；要啟用權控需在角色頁指派團隊、並於紀錄編輯時標上團隊。
- **MyTask 既有 `Category`（單值必填）** 與新的 `Categories`（多值標籤）並存，UI 標籤欄位以「標籤分類」標示以資區別。
- SqlServer migration 仍未 bootstrap（延續階段一決策），本階段僅產生 SQLite migration。
