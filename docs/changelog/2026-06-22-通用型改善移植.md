# 移植母專案通用型改善（0.4.2）

## 變更目的

階段三：從母專案 KnowledgeExtraction.AI 挑選**通用（非 AI）**的基礎/UX 改善移植回腳手架，共 4 項；皆低/中風險、無資料庫變更。

## 本次內容

1. **SignalR 訊息上限**：`Program.cs` 的 `AddInteractiveServerComponents()` 串接 `.AddHubOptions(o => o.MaximumReceiveMessageSize = 10 * 1024 * 1024)`（10MB），避免長文字表單同步時 circuit 中斷、畫面閃爍。
2. **Circuit 生命週期日誌**：新增 `MyProject.Web/Components/ApplicationCircuitHandler.cs`（繼承 `CircuitHandler`，記錄 circuit 開啟/連線起落/關閉；連線中斷以 Warning），於 `ServiceCollectionExtensions.AddApplicationServices` 註冊。
3. **CrudActionButton 圖示操作欄**：新增共用元件 `MyProject.Web/Components/Commons/CrudActionButton.razor`(+`.razor.css`)，以 `<MaterialIcon>` 呈現 edit/delete 圖示；將 7 個檢視（Project/MyTask/Meeting/RoleView/MyUser/Category/Team）操作欄的 emoji 按鈕（✏️/🗑️）統一改為此元件。
4. **Menu 圖示驗證測試**：新增 `MyProject.Tests/MenuIconTests.cs`，驗證 `Menu.json` 所有 icon 名稱皆在「有效 classic Material Icons」允許清單內，防止未來用到 Material Symbols 專有名稱造成破圖。

## 注意事項

- 既有 `btn-edit` / `icon-edit` 樣式於改用元件後可能成為孤兒，屬既有樣式，本次保留不刪。
- 示範檔 `RoleViewViewSample.razor` 非正式頁面，未一併改動。
- 排除項目（不在本次）：側邊欄收合 hover 修正（腳手架未見此 flyout 問題）、Modal 放大版全域樣式 90vw×90vh（對簡單 CRUD 表單偏大）、母專案 AI/外部整合相關日誌（非通用）。
