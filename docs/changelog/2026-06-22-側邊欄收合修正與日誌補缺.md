# 側邊欄收合飛出 hover 修正與日誌補缺（0.4.3）

## 變更目的

1. **修正側邊欄收合時飛出子選單提前消失**：收起側邊欄後，滑鼠在子選單圖示與飛出視窗之間移動時，飛出會提前關閉，難以點到子項。
2. **補齊日誌涵蓋率的少數缺口**：使用者要求全專案確保動作皆有日誌；經完整稽核，腳手架日誌覆蓋已相當完整，僅補 2 處缺口。

## 側邊欄修正（移植母專案 changelog 0.2.53 做法）

根因：腳手架收合狀態原本使用 AntDesign `<Menu Mode="Vertical">`（RC-Menu 浮層），圖示與浮層之間有間隙、無 hover 橋接，滑鼠跨越間隙即關閉。

做法（與母專案一致）：收合狀態改用**自訂飛出子選單**——
- `NavMenu.razor`：`IsSidebarCollapsed` 時渲染自訂 `collapsed-sidebar-menu`（`collapsed-menu-trigger` 圖示 + `collapsed-menu-flyout` 飛出）；展開狀態維持 AntDesign `Menu Mode="Inline"` 不變。
- `NavMenu.razor.cs`：新增 `GetCollapsedItemClass`/`GetCollapsedFlyoutItemClass`/`IsMenuKeyActive`/`GetMaterialIconKind(item)`；移除不再使用的 `GetMenuMode`/`GetMenuRenderKey`。
- `NavMenu.razor.css`：新增收合飛出樣式，含透明 `::before` 橋接（`left:-0.75rem;width:0.75rem`）維持 hover；顯示條件採 `:hover` 與 `:focus-visible`（**不含 `:focus-within`**，避免點擊後黏住）；移除收合狀態下已無用的 AntDesign `.ant-menu-*` 樣式。
- `NavMenu.razor.css`：補上 `.nav-scrollable-collapsed { overflow: visible; }`（`@media (min-width:641px)`）。否則 `.nav-scrollable` 的 `overflow-y:auto` 會把向右展開的飛出裁切，導致飛出完全不顯示。

## 日誌補缺

- `Components/Views/Commons/SplashView.razor.cs`：補注入 `ILogger<SplashView>`，記錄認證檢查（Debug 進站、Information 成功導向、Warning 未通過）。
- `Controllers/AuthController.cs` `Login`：成功發 token 後補一筆 `LogInformation`（帳號、UserId）。
- 全專案 `catch` 掃描確認無無記錄的 catch。

> 稽核結論：腳手架其餘部分（服務/控制器/檢視/認證/啟動/中介軟體）日誌已依 `日誌與設定檔說明.md §2` 的等級慣例完整覆蓋，本次不新增雜訊日誌。
