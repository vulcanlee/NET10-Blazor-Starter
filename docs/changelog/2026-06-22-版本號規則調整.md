# 版本號規則調整為「每次異動 Patch +1」（0.4.1）

## 變更目的

統一並簡化版本號遞增規則：**每次提交異動一律將版號最後一碼（Patch）+1**（例：`0.4.0 → 0.4.1`），不進位（`0.4.9 → 0.4.10`），不再依異動性質區分 Minor/Major。版號顯示格式維持 `Major.Minor.Patch (YYYY/MM/DD)`。

此規則**完全取代**先前的語義化遞增規則（bug→Patch、功能→Minor、不相容→Major）。`Major`／`Minor` 僅在明確決定「里程碑版本」時才人工調整。

## 本次修改檔案

- `docs/operations/維護規範.md`：§1.2 移除語義化表格，改為單一「Patch +1、不進位」規則；§5 範例同步。
- `readme.md`：第 8 節第 1 點補明確範例並移除依性質語意。
- `docs/architecture/開發慣例與限制速查.md`：§8 改為「一律 Patch +1」。
- `CLAUDE.md`、`AGENTS.md`：版本規則明確化為「最後一碼 +1」。
- `appsettings.json`：`SystemVersion` `0.4.0 → 0.4.1`（即套用新規則）。
