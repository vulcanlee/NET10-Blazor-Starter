# 維護規範

## 文件目的

定義本專案在「每次提交異動」時必須遵循的最低規範，包含版本遞增、文件同步、檔案編碼，以及 commit 前的自我檢查清單。

> 任何 PR 或 commit 在進入主線前，請先逐項對照本文件確認。

---

## 1. 規範一：每次更新必須 bump 版本

唯一版本來源是：

```
src/MyProject/MyProject.Web/appsettings.json
└─ SystemSettings.SystemInformation.SystemVersion
```

### 1.1 版號格式

```
Major.Minor.Patch (YYYY/MM/DD)
```

例：`0.1.39 (2026/05/08)`

### 1.2 遞增規則

| 異動性質 | 應遞增的位元 |
|----------|---------------|
| 一般 bug 修正、文件更新、UI 微調、欄位細節 | `Patch` +1 |
| 新增功能、新 API、新 Entity、新增頁面 | `Minor` +1，`Patch` 歸 0 |
| 不相容性變更、資料庫結構大改、整體架構重組 | `Major` +1，`Minor` 與 `Patch` 歸 0 |

### 1.3 日期同步

括號內日期必須更新為 commit 當天日期，格式 `YYYY/MM/DD`，與 `Asia/Taipei` 時區的當天一致。

### 1.4 檢查方式

```powershell
Select-String -Path "src/MyProject/MyProject.Web/appsettings.json" -Pattern "SystemVersion"
```

## 3. 規範三：UTF-8 編碼

### 3.1 適用範圍

- 所有 `.md`、`.razor`、`.cs`、`.json`、`.css` 文件。

### 3.2 推薦設定

- **建議使用「UTF-8 無 BOM」**，與 .NET 工具鏈與多數編輯器相容性最佳。
- 編輯後請確認繁體中文未出現 `?`、亂碼方塊或其他不可讀字元。

### 3.3 PowerShell 自我驗證

```powershell
Get-ChildItem -Path docs,readme.md -Recurse -Include *.md | ForEach-Object {
    $bytes = [IO.File]::ReadAllBytes($_.FullName)
    $hasBom = $bytes.Length -ge 3 -and $bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF
    "{0,-50}  bytes={1,-8} bom={2}" -f $_.Name, $bytes.Length, $hasBom
}
```

> 對於 PowerShell 自身建立的檔案（例如 `Out-File`、`Set-Content`），請務必加上 `-Encoding utf8` 旗標，避免預設 UTF-16 LE。

## 4. Commit 前自我檢查清單

請於 commit 前逐項確認（建議貼到 PR 描述中）：

- [ ] `appsettings.json` 的 `SystemVersion` 已遞增（含日期）。
- [ ] 本次異動所影響的 docs 文件已同步更新（對照第 2 節）。
- [ ] 所有新增 / 修改檔案皆採 UTF-8（無亂碼）。
- [ ] 新 / 改 API：已寫 `Debug` 進站、`Information` 成功、`Warning` 失敗、`Error` 例外四類日誌。
- [ ] 新 / 改 Entity：已執行 `Add-Migration` 並通過 `Update-Database`。
- [ ] 新 / 改 Razor 元件：已驗證 Sidebar 權限項目（若需要）。
- [ ] 本機 `dotnet build src/MyProject/MyProject.Web/MyProject.Web.csproj` 成功。
- [ ] 文件相對連結可正確開啟（`docs/` 內互引、與 `readme.md` 之間皆通）。

## 5. 異動範例

> 假設本次新增了「Issue（議題追蹤）」實體：

1. 新增 `MyProject.AccessDatas/Models/Issue.cs`。
2. 新增 Migration → 更新 [資料模型與資料庫.md](資料模型與資料庫.md)、[EFCore.md](EFCore.md)。
3. 新增 `IssueController`、DTO → 更新 [Web API 設計慣例.md](Web%20API%20設計慣例.md)。
4. 新增 `Menu.json` 節點與權限 → 更新 [認證授權與權限機制.md](認證授權與權限機制.md)。
5. 若有附件 → 更新 [檔案上傳機制.md](檔案上傳機制.md)。
6. 主 [readme.md](../readme.md) 第 4、6 節（功能列表 / 結構樹）若改變需同步。
7. `appsettings.json`：`SystemVersion` 由 `0.1.x` → `0.2.0`（屬於新功能 → Minor +1，Patch 歸 0），日期更新為 commit 當天。
