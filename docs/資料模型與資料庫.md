# 資料模型與資料庫

## 文件目的

說明本專案的資料庫採用方案、`BackendDBContext` 結構、各 Entity 與彼此關聯，以及 Migration 與啟動 Seed 的行為。

---

## 1. 資料庫採用方案

- **預設**：SQLite，連線字串由 `MagicObjectHelper.GetSQLiteConnectionString(databasePath)` 動態組成；`databasePath` 來自 `appsettings.json` 之 `SystemSettings.ExternalFileSystem.DatabasePath`（預設 `C:\temp\MyProject\DB`）。
- **替代**：SQL Server LocalDB；`appsettings.json` 已保留 `DefaultConnection` 字串，但目前 `Program.cs` 僅啟用 `UseSqlite(...)`，需要時請於 `Program.cs:150-152` 切換。
- **Collation**：`Chinese_Taiwan_Stroke_CI_AS`（於 `BackendDBContext.OnModelCreating` 統一指定）。

## 2. `BackendDBContext`

檔案：[`src/MyProject/MyProject.AccessDatas/BackendDBContext.cs`](../src/MyProject/MyProject.AccessDatas/BackendDBContext.cs)

### 2.1 DbSet 一覽

| `DbSet` | Entity | 用途 |
|---------|--------|------|
| `MyUser` | `MyUser` | 使用者帳號 |
| `MyTas` | `MyTask` | 工作項目（DbSet 名稱為 `MyTas`，Entity 名稱為 `MyTask`，係沿用早期命名） |
| `MyTasFile` | `MyTasFile` | 工作項目附件 |
| `Meeting` | `Meeting` | 會議紀錄 |
| `MeetingFile` | `MeetingFile` | 會議附件 |
| `Project` | `Project` | 專案 |
| `ProjectFile` | `ProjectFile` | 專案附件 |
| `RoleView` | `RoleView` | 角色（含權限 JSON） |

### 2.2 全域刪除政策

`OnModelCreating` 預設將所有外鍵的 `DeleteBehavior` 設為 `Restrict`：父資料表若仍有子資料時不可被強制刪除。

### 2.3 Cascade 例外

下列三組「主表 → 附件表」改為 `DeleteBehavior.Cascade`，主表刪除時連動清除附件紀錄：

```csharp
Project    → Project.Files     → ProjectFile.ProjectId
MyTask     → MyTask.Files      → MyTasFile.MyTasId
Meeting    → Meeting.Files     → MeetingFile.MeetingId
```

> 注意：附件「資料表紀錄」會被 EF Core Cascade 處理；但**實體檔案**仍須由 Service 層在刪除前後手動移除。詳見 [檔案上傳機制](檔案上傳機制.md)。

## 3. 主要 Entity 概觀

| Entity | 主要欄位 | 備註 |
|--------|----------|------|
| `MyUser` | `Account`、`Name`、`Email`、`Password`、`Salt`、`IsAdmin`、`Status`、`RoleViewId` | 密碼以 `PasswordHelper.GetPasswordSHA(salt, password)` 雜湊；`RoleViewId` 連結到角色。 |
| `RoleView` | `Name`、`TabViewJson` | `TabViewJson` 為 `List<List<string>>` 序列化後的權限樹，對應 Sidebar 二維結構。 |
| `Project` | `Title`、`Description`、`StartDate`、`EndDate`、`Status`、`Priority`、`CompletionPercentage`、`Owner`、`CreatedAt`、`UpdatedAt`、`Files` | 日期欄位可為 null。 |
| `MyTask` | `Title`、`Description`、`StartDate`、`EndDate`、`Category`、`Status`、`Priority`、`CompletionPercentage`、`Owner`、`ProjectId`、`Files`、`CreatedAt`、`UpdatedAt` | 透過 `ProjectId` 關聯到 `Project`。 |
| `Meeting` | `Title`、`Description`、`Summary`、`Participants`、`ProjectId`、`Files`、`CreatedAt`、`UpdatedAt` | 透過 `ProjectId` 關聯到 `Project`。 |
| `ProjectFile` / `MyTasFile` / `MeetingFile` | `FileName`、`StoredPath`、`Size`、外鍵欄位 | 一筆紀錄對應一份實體檔案。 |

實際欄位請以 `src/MyProject/MyProject.AccessDatas/Models/` 內的程式碼為主。

## 4. Migration 與啟動 Seed

### 4.1 Migration 檔案

目前僅一份初始 Migration：`src/MyProject/MyProject.AccessDatas/Migrations/20260316083639_init.cs`。

新增 Migration、移除 Migration、套用資料庫的指令請見 [EFCore.md](EFCore.md)。

### 4.2 啟動時自動處理

`Program.cs:175-258` 會在 `app.Build()` 後執行：

1. 若有任何 Migration → `dbContext.Database.Migrate()`。
2. 否則 → `dbContext.Database.EnsureCreated()`。
3. **Seed 預設角色**：`RoleView.Name == MagicObjectHelper.預設角色`（不存在則新增），其 `TabViewJson` 來自 `RolePermissionService.GetRolePermissionAllNameToJson()`。
4. **Seed 預設使用者**：`MyUser.Account == MagicObjectHelper.開發者帳號`（不存在則新增），密碼採同帳號值 + Salt SHA 雜湊，並關聯到上一步的 `RoleView`。

> 啟動 Seed 為冪等行為：每次啟動會重新覆寫預設角色的 `TabViewJson`，並重置 `IsAdmin = true`，確保開發者帳號永遠可登入。

- 對應的 commit 紀錄為 `MyTas 全面更名為 MyTask 並重建 migration`（commit 05ab875）。
