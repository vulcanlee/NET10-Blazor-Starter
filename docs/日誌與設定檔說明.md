# 日誌與設定檔說明

## 文件目的

整理本專案使用 NLog 寫入日誌的方式、不同層級的使用時機，以及 `appsettings.json` 各區段的完整說明。

---

## 1. 日誌方案：NLog

採 `NLog.Web.AspNetCore 6.1.2`，於 [`src/MyProject/MyProject.Web/Program.cs:33-56`](../src/MyProject/MyProject.Web/Program.cs) 完成設定：

1. 從 `appsettings.json` 讀取 `NLog.BasePath`（預設 `C:\temp\Logs`）。
2. 在 `BasePath/MyProject.Web/` 下建立日誌目錄。
3. 設定 NLog Internal Logger 寫入 `MyProject.Web-nlog-internal.log`。
4. 將 `BasePath` 與 `LogFilenamePrefix` 變數寫入 NLog 設定，供 `nlog.config` 內 `${var:BasePath}` 引用。
5. `builder.Logging.ClearProviders()` 清除預設 Provider，由 `builder.Host.UseNLog()` 接管。

`nlog.config` 位於 `MyProject.Web/` 根目錄，定義具體 Target / Rule。

## 2. 何時使用何種等級

| 等級 | 使用時機 | 範例 |
|------|----------|------|
| `Trace` | 細節最高的逐步追蹤；通常不啟用。 | 內部演算法每一步狀態。 |
| `Debug` | 流程關鍵節點、輸入參數、查詢條件。 | `logger.LogDebug("Received task search request. Keyword={Keyword}", request.Keyword);` |
| `Information` | 商業流程成功完成、啟動 / 關閉、Seed 完成。 | `logger.LogInformation("Application startup completed.");` |
| `Warning` | 找不到資源、業務驗證未通過、可恢復例外。 | `logger.LogWarning("Task get request could not find record. TaskId={TaskId}", id);` |
| `Error` | 例外、外部資源失敗、重要流程中斷。 | `logger.LogError(ex, "Failed to search tasks");` |
| `Critical` | 系統幾乎無法繼續運作。 | DB 連線完全失敗、執行階段崩潰。 |

撰寫原則：

- 訊息以英文 + 結構化參數（`{Name}` 佔位）為主，方便日後檢索。
- 使用者面向訊息（中文）保留在 `ApiResult.Message` / `Notification`，不要混入日誌訊息。
- 例外請使用 `logger.LogError(ex, "...")` 寫入完整堆疊。

## 3. 全域請求日誌

`Program.cs:277-305` 註冊一個自訂 middleware：

- 對每筆 HTTP 請求記錄 `Method`、`Path`、`StatusCode`、`ElapsedMilliseconds`。
- 例外發生時改寫為 `Error`，並重新拋出讓上層 ExceptionHandler 處理。

由此可在不額外加程式的情況下取得每個 endpoint 的響應耗時。

## 4. `appsettings.json` 區段全表

檔案：[`src/MyProject/MyProject.Web/appsettings.json`](../src/MyProject/MyProject.Web/appsettings.json)

```jsonc
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*",
  "NLog": { "BasePath": "C:\\temp\\Logs" },
  "SystemSettings": {
    "ConnectionStrings": {
      "DefaultConnection":         "...",
      "SQLiteDefaultConnection":   "Data Source=BackendDB.db"
    },
    "SystemInformation": {
      "SystemName":        "Blazor開發啟動範本專案",
      "SystemDescription": "Blazor開發啟動範本專案",
      "SystemVersion":     "0.1.39 (2026/05/08)"
    },
    "ExternalFileSystem": {
      "DatabasePath":      "C:\\temp\\MyProject\\DB",
      "DownloadPath":      "C:\\temp\\MyProject\\Download",
      "UploadPath":        "C:\\temp\\MyProject\\Upload",
      "ProjectFilePath":   "C:\\temp\\MyProject\\ProjectFile",
      "TaskFilePath":      "C:\\temp\\MyProject\\TaskFilePath",
      "MeetingFilePath":   "C:\\temp\\MyProject\\MeetingFilePath"
    }
  },
  "AutoMapper:LicenseKey": ""
}
```

### 4.1 `Logging`

.NET 預設記錄等級。NLog 啟動後會接管實際輸出，但此區段仍影響 `Microsoft.*` 等內建類別的 Filter。

### 4.2 `NLog.BasePath`

NLog 寫檔的根目錄；專案會在其下建立 `MyProject.Web` 子目錄。若不存在會被 NLog 自動建立。

### 4.3 `SystemSettings.ConnectionStrings`

| 欄位 | 用途 |
|------|------|
| `DefaultConnection` | SQL Server LocalDB 連線（目前未啟用，保留供未來切換）。 |
| `SQLiteDefaultConnection` | SQLite 範本字串；實際連線字串由 `MagicObjectHelper.GetSQLiteConnectionString` 結合 `DatabasePath` 動態產生。 |

### 4.4 `SystemSettings.SystemInformation`

| 欄位 | 用途 |
|------|------|
| `SystemName` | 顯示在 UI 上的系統名稱。 |
| `SystemDescription` | 顯示用描述。 |
| **`SystemVersion`** | **唯一版本來源**，每次更新必須遞增；格式 `Major.Minor.Patch (YYYY/MM/DD)`。詳見 [維護規範](維護規範.md)。 |

### 4.5 `SystemSettings.ExternalFileSystem`

| 欄位 | 用途 |
|------|------|
| `DatabasePath` | SQLite 資料庫檔放置目錄。 |
| `DownloadPath` | 對應 `/UploadFiles` 的靜態資源目錄。 |
| `UploadPath` | 通用上傳暫存目錄。 |
| `ProjectFilePath` | 專案附件根目錄（依年/月分層）。 |
| `TaskFilePath` | 工作附件根目錄。 |
| `MeetingFilePath` | 會議附件根目錄。 |

啟動時 `Program.cs:132-141` 會逐一呼叫 `EnsureDirectoryExists` 建立缺少的目錄。

### 4.6 `AutoMapper:LicenseKey`

AutoMapper 12+ 商業授權金鑰；留空會走免費條款。註冊位置：`Program.cs:113-122`。

## 5. 環境差異設定

`appsettings.Development.json` 僅覆寫 Logging：

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  }
}
```

未來可在 `appsettings.Production.json` 內切換 SystemSettings（例如連線字串、外部目錄）以符合不同部署環境。
